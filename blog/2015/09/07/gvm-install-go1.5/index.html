gvmでgo1.5をインストールする

Go1.5をgvmで入れようとしたらちょっと躓いた話

使うもの

  • gvm(Golang Version Manager)

さっそく入れてみる

$ gvm install go1.5
Installing go1.5...
 * Compiling...
ERROR: Failed to compile. Check the logs at /Users/t_murano/.gvm/logs/go-go1.5-compile.log
ERROR: Failed to use installed version

…あれ？なんかコンパイルに失敗してるし。
とりあえずログを見てみる。

$ less /Users/t_murano/.gvm/logs/go-go1.5-compile.log
##### Building Go bootstrap tool.
cmd/dist
ERROR: Cannot find /Users/t_murano/go1.4/bin/go.
Set $GOROOT_BOOTSTRAP to a working Go tree >= Go 1.4.
./make.bash: line 121: /Users/t_murano/go1.4/bin/go: No such file or directory

あ、そういえばGo1.5からBootstrap化されたんだっけ。
Bootstrap用に必要なGo(1.4以上)のパスを設定する環境変数があるので、とりあえずgvmで入れてあったGo1.4.2を指定してみる。

$ export GOROOT_BOOTSTRAP=/User/t_murano/.gvm/gos/go1.4.2
$ gvm install go1.5
Installing go1.5...
 * Compiling...
$ gvm list

gvm gos (installed)

=> go1.4.2
   go1.5
   system

お、入った。
別にgvm関係ないんだけど、GOROOT_BOOTSTRAPっていう環境変数を設定しないとGo1.5をコンパイル出来ないという話でした。
