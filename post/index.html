gvmでgo1.5をインストールする

Go1.5をgvmで入れようとしたらちょっと躓いた話 使うもの gvm(Golang Version Manager) さっそく入れてみる $ gvm install go1.5 Installing go1.5... * Compiling... ERROR: Failed to compile. Check the logs at /Users/t_murano/.gvm/logs/go-go1.5-compile.log ERROR: Failed to use installed version …あれ？なんかコンパイルに失敗してるし。 とりあえずログを見てみる。 $ less /Users/t_murano/.gvm/logs/go-go1.5-compile.log ##### Building Go bootstrap tool. cmd/dist ERROR: Cannot find /Users/t_murano/go1.4/bin/go. Set $GOROOT_BOOTSTRAP to a working Go tree >= Go 1.4. ./make.bash: line 121: /Users/t_murano/go1.4/bin/go: No such file or directory あ、そういえばGo1.5からBootstrap化されたんだっけ。 Bootstrap用に必要なGo(1.4以上)のパスを設定する環境変数があるので、とりあえずgvmで入れてあったGo1.4.2を指定してみる。 $ export GOROOT_BOOTSTRAP=/User/t_murano/.gvm/gos/go1.4.2 $ gvm install go1.5 Installing go1.5...
Tue, Aug 25, 2015

YAPC::Asia Tokyo 2015に参加した話

YAPC::Asiaに参加してきたのですが、どうやらブログを書くまでがYAPCとのことなのでブログ作っちゃいました。 どのトークやイベントも非常に面白かったのですが、個人的に印象的だった部分だけピックアップします。 TBD (Matz) RubyのパパことMatzのトーク まずトークの内容以前にスライド内で競争してるウサギとカメが気になって仕方なかった。 Rabbitというプレゼンテーションツールを使用していたみたい。 内容はRubyディスやStreemという開発中の言語についてなど。 「絶対に型を書きたくないでござる」発言が最高に面白かった(もしかしてMatzはCharlotte観てたりするのかな？) 懇親会 とにかく人が多かった。 突然GMOペパボのあんちぽさんに声をかけられたりなど、思いもよらないイベントが… 自分はだんごゆっけの平和な話というPodcastのファンだったので、勇気を出してゆーすけべーさんに声を掛けてみたら快くおしゃべりに応じてくれて超ハッピー(だんごさんは残念ながら見つけられなかった) YAPCあるある 先ほどのだんごゆっけのお二人が司会で、これまでのYAPC::Asiaを主催してきた方々を迎えての振り返り談義 最初の拍手での入場や、941さんとmakiさんのツッコミが面白かったりなど、バラエティ番組みたいなノリになってた気がする。 「ACTのバグはフランスの作者の家に行って泊まって直した」のくだりは流石のmiyagawaさん 自分は初参加だったが、これまで10年間の積み重ねで今回があったと思うと感慨深いものがある。 Profiling & Optimizing in Go Golang開発者であるBrad Fitzpatrick氏のトーク Camel + Gopher => Campher is キモい ライブコーディングを交えつつGoのデータ構造や最適化手法を丁寧に解説していてとてもためになった。 pprofなどのツールを全く知らなかったのでデモ用の資料で復習しよう まとめ そんな感じでとっても充実した2日間でした。 参加者の方々とまた何かしらのイベントで会えたらいいなー、と思いつつこれで私のYAPCは終わりとなります。 スタッフやスピーカーや参加者の皆様方、ありがとうございました！
